# Flashcard 复习卡片生成 Skill

从各种格式文件中提取知识内容，自动生成双语/名词解释闪卡

---

## 1. 输入处理

### 1.1 用户输入方式

用户可以通过以下方式提供素材：

- **方式A**：提供文件路径（支持多种格式），如 `/flashcard C:\path\to\file.txt`
- **方式B**：直接粘贴文本内容，如 `/flashcard cell:细胞，nucleus:细胞核`
- **方式C**：混合多个文件和一个描述，如 `/flashcard chapter1.pdf chapter2.pptx --mode 英译汉`

### 1.2 支持的文件格式与提取方法

| 格式 | 提取方式 |
|------|---------|
| `.txt` | 直接用 `Read` 工具或 `Get-Content` 读取 |
| `.pdf` | 用 `Read` 工具（pages参数）或 python pdfplumber/PyMuPDF 提取 |
| `.docx` | 用 `python-docx` 提取文本 |
| `.pptx` | 用 `python-pptx` 提取幻灯片文本 |
| `.jpg` / `.png` | 用 `Read` 工具（图片会直接呈现），结合文字描述提取内容 |

**图片处理说明**：对于 jpg/png 格式，Read 工具会展示图片内容。观察图片中的文字信息并提取为可用的卡片数据。

### 1.3 提取原则

- 保留章节/段落结构
- 识别专业术语和对应释义/翻译
- 去重合并同类项
- 对于 PPT，每个幻灯片标题和正文作为一组内容

## 2. 内容分析阶段

### 2.1 自动识别卡片模式

分析提取到的文本内容，自动判断适合哪种卡片模式：

**英译汉模式**（检测到大量英文术语 + 中文对照时）：
- 格式类似：`term → 翻译`、`term: 翻译`、`term 翻译`
- 或表格中的英中对照列
- 每张卡片正面为英文术语，背面为中文翻译

**名词解释模式**（检测到中文术语 + 长段释义时）：
- 格式类似：`名词：解释……`、`术语——定义`
- 或教材/讲义中的概念定义
- 每张卡片正面为术语/概念名，背面为详细释义

**混合模式**（两种都有时）：
- 按 section 分区，生成同时包含 en-cn 和 cn-def 两种卡片的HTML

### 2.2 智能解析规则

从文本中解析卡片对时，按以下优先级匹配：

1. **显式分隔符**：`→` `←` `:` `：` `-` `—` `\t` `|` 分隔的成对内容
2. **枚举列表**：编号或项目符号列表中的成对条目
3. **段落对**：交替出现的术语和定义段落
4. **表格结构**：Markdown 表格或文本表格的两列
5. **自然段落**：单段话中的核心名词+定义描述

## 3. 卡片生成阶段

### 3.1 HTML 结构

**核心UI元素**：
- 暗色主题UI，使用CSS变量定义配色（`--bg`, `--card-bg`, `--text`, `--accent`, `--correct`, `--wrong` 等）
- 四按钮动作区：`⬆ 上一个` · `😕 忘了` · `🔄 翻转` · `✅ 记得`（`上一个`在最左，`翻转`居中偏右）
- 底部工具栏：`🔀 打乱重来` · `📝 复习遗忘词` · `📋 词汇表` · `🔄 全部重置`
- 三个统计卡片：**剩余**（蓝色） · **已掌握**（绿色） · **待复习**（黄色）
- 卡片索引显示：`第 X / Y 个`
- 进度条：根据已掌握比例更新
- 分类标签（section pills）：切换不同主题/模式
- 操作提示（header subtitle）：显示键盘快捷键提示
- 上一个词回溯：基于 `historyStack` 栈实现

**复习算法**（队列+随机洗牌+历史栈）：
1. 当前分组的全部词索引加入队列，随机打乱
2. 用户逐张复习，**记得** → 加入已掌握集合；**忘了** → 加入待复习集合并追加到队列末尾；每次标记时当前状态推入 `historyStack`
3. 当队列耗尽时，若有遗忘词则重新打乱形成新队列继续复习
4. 全部掌握后显示「全部复习完毕」
5. **回溯**：`historyStack` 记录每次标记操作，点击"上一个"或按 `↑` 可撤销上一步标记

**键盘快捷键**：
- `Space` / `Enter`：翻转卡片
- `←` (ArrowLeft)：标记忘了
- `→` (ArrowRight)：标记记得
- `↑` (ArrowUp)：回溯上一个词
- `R`：全部重置
- `F`：复习遗忘词

### 3.2 卡片数据格式

```javascript
// 英译汉模式
{ front: "English Term", back: "中文翻译", section: "en-cn" }

// 名词解释模式
{ front: "术语名称", back: "详细释义内容", section: "cn-def" }
```

### 3.3 变量声明规范

生成的HTML中的 JavaScript 使用 `let`/`const` 声明变量，现代浏览器均完全支持。

### 3.4 卡片背面渲染

所有卡片的正反面统一使用 `textContent` 赋值，确保安全性和性能。

### 3.5 数据注入安全转义（重要！）

将解析出的卡片数据嵌入 HTML 的 `<script>` 时，必须对每一条数据的 `front` 和 `back` 字段执行以下转义：

| 原始字符 | 替换为 | 原因 |
|---------|--------|------|
| `\` (反斜杠) | `\\` | JS字符串转义符 |
| `"` (ASCII双引号) | `\"` 或替换为 `「」` | 与JS字符串定界符冲突 |
| 模板字面量 `${` | `\${` 或改用字符串拼接 | 误触模板插值语法 |
| 换行符 `\n` | `\n` (转义) 或 `\\n` | 不能有物理换行在字符串内 |
| `<` 和 `>` | `&lt;` / `&gt;` | 防止HTML注入 |
| `&` | `&amp;` | HTML实体安全 |
| 控制字符（U+0000-U+001F） | 移除或替换为空格 | 不可见字符破坏结构 |

### 3.6 HTML 标题设定

- 如果输入是文件名，从文件名提取学科/主题名
- 如果输入是文本内容，从内容推断主题
- 标题格式：`{主题} · 复习卡片`

### 3.7 文件命名

- 命名格式：`{主题}-flashcards.html`
- 如果文件已存在，询问用户是否覆盖

### 3.8 词汇表增强功能

**搜索过滤**：
- 搜索输入框为**静态HTML**（直接写在模态框中，不参与动态渲染）
- 使用全局事件监听（`addEventListener('input', ...)`）而非 `oninput` 属性
- placeholder 为 `🔍 搜索词汇...`，设置 `autocomplete="off"`
- 支持实时搜索，同时匹配 front 和 back，不区分大小写
- 打开模态框后自动聚焦搜索框

**背诵状态显示**：
- 每个词汇行左侧添加状态指示器：
  - `✅` 已掌握（绿色）
  - `🔄` 待复习（黄色）
  - `○` 未学习（灰色）

**词汇表排版**：
- 三列布局：状态 | 术语（front）| 释义（back）
- 使用 CSS Grid 或 Flex 实现，状态列固定宽度（40px），术语和释义列自适应
- 已掌握的词汇行使用更低透明度的背景
- 词汇表顶部显示统计摘要
- 词汇行容器 `max-height: 55vh`，超出滚动

## 4. 输出规范

### 4.1 语言风格规则（重要！）

- **所有输出给用户的句子末尾添加"喵"**
- **所有句号"。"替换为"~"**
- 代码内容、文件路径、HTML中的文本内容不受此规则影响
- 此规则用于方便判断skill是否被正常调用以及上下文统一

### 4.2 输出示例

```
✅ 已成功从文件中提取到 32 个卡片对喵~
📋 其中英译汉 20 个，名词解释 12 个喵~
📁 HTML 已生成至: C:\xxx\生物化学-flashcards.html喵~
💡 用浏览器打开即可开始复习喵~ (空格翻转，左右箭头标记记得/忘了)喵~
```

## 5. 工作流程

### 5.1 步骤概览

```
Step 1: 接收输入 → 识别文件格式 → 提取文本
Step 2: 分析文本 → 识别模式 → 解析卡片对
Step 3: 转义清洗 → 生成 HTML → 写入文件
Step 4: 语法自检 → 通过后 → 报告结果
```

### 5.2 详细流程

**Step 1 - 文本提取**：
1. 确认文件存在
2. 根据文件格式选择合适的提取方法
3. 对于图片文件（jpg/png），使用Read工具查看后提取文字信息
4. 将提取的文本合并为一个字符串

**Step 2 - 卡片解析**：
1. 分析文本结构，判断卡片模式（英译汉/名词解释/混合）
2. 按规则逐条解析卡片对
3. 去重（相同front只保留第一个）
4. 如果卡片数量为0，提示用户并提供手动输入建议

**Step 3 - HTML生成**：
1. 构建完整的HTML文件，包含所有交互功能
2. 嵌入解析出的卡片数据前，按安全转义规则对每条数据做处理
3. 将完整 HTML 写入文件

**Step 4 - 自检调试（必须执行！）**：
1. 用 Node.js 验证生成的 HTML 中的 JavaScript 语法
   ```powershell
   node -e "const fs=require('fs');const h=fs.readFileSync('输出文件路径','utf8');const m=h.match(/<script>([\s\S]*?)<\/script>/);if(m)try{new Function(m[1]);console.log('✅ JS语法正确')}catch(e){console.log('❌ '+e.message)}"
   ```
2. 如果语法检查报错：定位错误行，检查是否有未转义的特殊字符，修复后重新生成
3. 确认无误后再向用户报告结果

## 6. 特殊情况处理

### 6.1 文件不存在或无法读取
- 提示用户检查路径，并列出当前目录下的文件供参考

### 6.2 提取内容为空
- 检查文件是否损坏或格式不支持

### 6.3 卡片数量极少（<3个）
- 正常生成HTML，提醒用户内容较少

### 6.4 图片文字模糊
- 将能识别的部分提取，标记不确定的内容

### 6.5 混合语言内容
- 智能拆分：英文为主的内容归入英译汉，中文术语+长释义归入名词解释

### 6.6 生成的 HTML 无法正常使用
- 首先用 Node.js 执行语法自检，定位 JavaScript 错误
- 常见原因：数据中的 ASCII 双引号、反斜杠、模板语法 `${` 未转义
- 修复后重新生成并再次验证

## 7. 技术参考

### 7.1 Python 依赖安装

```powershell
pip install python-docx python-pptx pdfplumber
```

### 7.2 HTML 模板关键特征

- css变量使用暗色主题（--bg: #0f172a, --card-bg: #1e293b）
- 卡片翻转使用CSS 3D transforms（rotateY）
- 数据存储在 `ALL_WORDS` 数组
- 交互逻辑：点击翻转、按钮标记记得/忘了、键盘快捷键
- 分类过滤：section-pills 切换不同卡片类型
- 进度统计：progress-bar + stats-bar
- 词汇表模态框：word-list modal（带搜索过滤、背诵状态显示、三列布局）
- 上一个词回溯：`historyStack` 数组 + `goBack()` 函数
- **卡片文字样式**：正面术语 `font-size: 1.8rem`，`font-weight: 800`；背面释义 `font-size: 1.15rem`，`line-height: 1.7`

## 8. 使用示例

### 示例1：从TXT文件生成英译汉卡片
```
/flashcard C:\notes\生化词汇.txt
```
输入内容如 `DNA: 脱氧核糖核酸` → 生成 `生化词汇-flashcards.html`

### 示例2：从PDF生成名词解释卡片
```
/flashcard C:\textbook\ch3.pdf --mode 名词解释
```

### 示例3：从PPTX生成混合卡片
```
/flashcard C:\slides\遗传学.pptx
```
自动识别英译汉和名词解释两种模式
