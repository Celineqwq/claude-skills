# 文献翻译 (Literature Translation)

将中英文学术论文逐段翻译，生成双语对照PPT（一页原文，一页译文交替排列）。

---

## 1. 输入处理

### 1.1 用户输入格式
用户可以通过以下方式提供论文：

- **方式A**：直接粘贴论文全文文本（推荐），可附带标题
- **方式B**：提供文件路径（支持 `.pdf`, `.docx`, `.txt`）
- **方式C**：在对话中已上传的文件或之前出现的文本

### 1.2 提取论文文本
- 如果是文件路径，使用 `python` 库提取文本：
  - PDF：`PyMuPDF` (fitz) 或 `pdfplumber`
  - DOCX：`python-docx`
  - TXT：直接读取
- 提取时保留段落结构和章节划分

### 1.3 提取论文元信息
从文本中提取：
- **论文标题**（Title）
- **作者**（Authors）——用于首页展示及确认人名，翻译时保留不译
- **期刊名称**（Journal）
- **导师姓名**（Mentor / Corresponding Author）——如有
- **DOI 编号**（DOI）
- **摘要**（Abstract）
- **章节结构**（Introduction, Methods, Results, Discussion, Conclusion 等）

---

## 2. 分析阶段：识别专业领域

分析论文内容确定其专业领域。方法：

1. **关键词扫描**：查找领域特有的术语和缩写
2. **期刊/会议来源**：如果用户提供了出处信息
3. **方法学特征**：统计方法 → 社科/医学；实验方法 → 理工科；语料库方法 → 语言学；等等
4. **参考文献分析**：引用文献的领域分布

**常见专业领域及对应关注点**：

| 领域 | 翻译要点 |
|------|---------|
| 医学/药学 | 解剖术语、疾病名称、药物名称使用标准译名（遵守中国药典/医学名词委员会标准） |
| 计算机科学 | 算法名、框架名不译，术语参考《计算机科学技术名词》 |
| 物理学/工程 | 单位符号保留，公式保留原格式，术语参考《物理学名词》 |
| 生物学/化学 | 化合物命名遵循IUPAC中文规则，物种名使用拉丁学名+中文名 |
| 法学 | 法律术语准确对应中国法律体系概念 |
| 经济学/管理学 | 专业术语参考《经济学名词》《管理科学技术名词》 |
| 语言学/教育学 | 术语参考对应学科名词审定委员会标准 |

**关键约束**：人名、地名、机构名、期刊名、品牌名一律保留原文不翻译。

---

## 3. 翻译阶段

### 3.1 翻译原则
1. **准确性优先**：专业术语必须准确，不确定时保留原文并加注
2. **严谨性**：数字、单位、公式、符号严格保留原样
3. **流畅性**：符合中文表达习惯，但不过度意译
4. **一致性**：同一术语在全文中保持翻译一致
5. **完整性**：不省略任何内容，包括作者 affiliation、基金信息等（但最终PPT不包含引用和致谢）

### 3.2 逐段翻译
将论文按章节/段落拆分为逻辑单元，逐单元翻译：

- 每个逻辑单元生成 **一对翻译**（原文 + 译文）
- 原文保留原格式（段落、编号、加粗等标记用文字描述）
- 公式用 LaTeX 格式或原样保留
- 表格用文字形式描述

### 3.3 内容分块策略
确定PPT每页的内容分块：

- **每个章节为一个单元**（Abstract → Introduction → Methods → ...）
- 如果一个章节文本很长（超过约 500 字中文或 800 词英文），拆分为多个子单元
- 每个子单元对应一页PPT（原文页 + 译文页为一组）
- 子单元需要一个小标题（如 "Introduction (1/3)" 或 "引言（第1部分）"）

## 4. PPT 生成阶段

### 4.1 准备工作
确认 `python-pptx` 已安装。如未安装，先执行：
```powershell
pip install python-pptx
```

### 4.2 生成 Python 脚本
编写一个完整的 Python 脚本用于生成 PPT，包含以下关键要素。

> **注意**：脚本的配置区需要填入从论文中提取的元信息（标题、作者、期刊、导师、DOI）。
> 如论文中缺少某个字段（如导师姓名或 DOI），对应变量值设为 `""` 即可，首页不会显示空行。

**PPT 格式规范**：

| 项目 | 要求 |
|------|------|
| 幻灯片尺寸 | 16:9 宽屏（13.333 × 7.5 英寸） |
| 首页 | 论文大标题居中（28-32号微软雅黑加粗），下方小字（14号灰色）分行展示作者、期刊、导师、DOI 等元数据 |
| 目录页 | "目录 / Table of Contents" 标题，下方列出各大章节（英文标题 + 中文缩进对照），自动滤除重复和子章节 |
| 内容页标题区 | **留空**（不填写PPT布局的标题占位符） |
| 内容页正文区 | 包含章节小标题 + 正文 |
| 章节小标题字体 | 20号微软雅黑，加粗 |
| 正文字体 | 18号微软雅黑 |
| 行距 | 约 28pt（1.5倍） |
| 对照方式 | 首页 → 目录页 → 英文 → 中文 → 英文 → 中文……交替排列 |
| 语言标识 | 每页顶部用灰色12号小字标注 "English" 或 "中文" |
| 不包含内容 | 引用文献列表、致谢、作者贡献声明等 |
| 文件名 | `{论文标题}文献翻译.pptx` |

### 4.3 执行生成

1. 将完整脚本写入临时文件（如 `C:\Users\QwQ\.claude\temp\gen_ppt.py`）
2. 在 PowerShell 中运行：`python "C:\Users\QwQ\.claude\temp\gen_ppt.py"`
3. 确认脚本执行成功，输出路径正确

## 5. 自检阶段

生成 PPT 后，执行以下检查：

### 5.1 排版检查（使用 Python 脚本验证）

编写并运行验证脚本检查字体、字号、页面对照结构等。

### 5.2 人工复核要点

1. ✅ **页面顺序正确**：首页 → 目录页 → 英文 → 中文 → 英文 → 中文...
2. ✅ **首页内容完整**：标题居中，下方有作者、期刊、导师、DOI 等元数据（小字灰色）
3. ✅ **目录页准确**：列出了论文的主要章节（去除了重复和子章节），双语对照
4. ✅ **翻译完整**：论文所有主要章节均已翻译，无遗漏
5. ✅ **术语一致**：同一术语在全文中翻译一致
6. ✅ **人名地名保留**：未翻译人名、地名、机构名
7. ✅ **无引用/致谢**：参考文献和致谢部分已排除
8. ✅ **文件名正确**：格式为 `{论文标题}文献翻译.pptx`

### 5.3 发现问题时的处理

- **字体问题**：检查 python-pptx 的 font.name 设置是否被 PowerPoint 覆盖
- **字号问题**：调整脚本中的字号常量重新生成
- **文本溢出**：将过长的段落拆分为多个子页，或略微缩小字号到 16pt
- **结构问题**：调整分块策略后重新生成

---

## 6. 特殊情况处理

### 6.1 论文非常短（只有摘要）
- 依然生成对照PPT，摘要在前
- 添加标注 "Short Paper / 短文"

### 6.2 论文非常长（超过20个章节）
- 每章的核心部分保留翻译
- 次要内容（如附录中的推导细节）可以摘要式处理

### 6.3 英文论文中含有中文引用
- 保持原文中的中文不变，但引用格式保持统一

### 6.4 中文论文翻译为英文
- 同样处理方式，方向反转：中文为原文，英文为译文

### 6.5 图文混排的论文
- 用方括号标注插图位置，如 `[Figure 1: 实验装置示意图]`
- 图表标题需要翻译，但图中文字保留原文并在译文中标注

### 6.6 专业术语不确定时
- 保留原文并加括号给出建议译名
- 在整篇翻译中保持一致性

**输入**：一篇关于 CRISPR-Cas9 基因编辑技术的英文论文

| 页码 | 类型 | 内容 |

| ... | ... | ... |

## 8. 注意事项

- 翻译的质量直接取决于对专业领域的准确识别和术语的准确使用
- 如果用户对特定术语有偏好的译法，遵循用户的要求
- 对于多义词（如 "cell" 在生物学中为"细胞"，在监狱学中为"牢房"），根据领域判断
- 生成完毕后，**向用户报告输出文件路径**，并告知自检结果
- 如果自检发现问题，**在报告时列出所有问题**，并提供修复方案
- 不要在没有用户确认的情况下覆盖已存在的同名文件
